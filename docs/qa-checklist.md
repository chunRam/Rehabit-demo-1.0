# QA Checklist

이 문서는 릴리스 전 수행해야 할 핵심 QA 항목을 요약합니다. 모든 팀원이 체크리스트를 기반으로 테스트 결과를 기록하고, 미해결 이슈가 있는 경우 QA 슬랙 채널로 공유합니다.

## 사전 준비
- [ ] 테스트 환경(.env.staging) 및 최신 마이그레이션이 적용되어 있는지 확인합니다.
- [ ] Sentry 및 Datadog 대시보드에서 오류/트레이스 수집 여부를 점검합니다.
- [ ] 테스트 계정과 초기 데이터 시드가 최신 상태인지 확인합니다(자세한 내용은 [테스트 데이터 시드 가이드](./test-data-seed.md) 참고).

## API (NestJS)
- [ ] `/healthz` 응답이 200을 반환하는지 확인합니다.
- [ ] `/ml/predict` 엔드포인트가 FastAPI 서비스와 정상적으로 통신하는지 확인합니다.
- [ ] 비정상 입력(잘못된 JSON 등)에 대해 4xx 에러를 반환하는지 확인합니다.

## ML 서비스 (FastAPI)
- [ ] 모델 예측 API `/predict`가 요청 본문을 검증하고 예상 JSON 스키마로 응답하는지 확인합니다.
- [ ] Datadog 트레이스에 예측 요청이 수집되는지 확인합니다.

## 모바일 앱 (React Native)
- [ ] 사용자 여정 시나리오 테스트(Detox/시뮬레이션)가 통과하는지 확인합니다.
- [ ] 신규 사용자 온보딩, 프로그램 선택, 활동 기록 흐름이 정상 동작하는지 수동으로 확인합니다.
- [ ] 빌드 산출물(안드로이드 APK/아이오에스 IPA)이 생성되는지 확인합니다.

## 회귀 테스트
- [ ] 핵심 유스케이스에 대한 스모크 테스트를 실행합니다.
- [ ] 최근 버그 수정과 관련된 회귀 테스트 케이스를 재검증합니다.

## 마무리
- [ ] 모든 체크 항목을 완료한 후 QA 결과를 기록하고 스테이크홀더에게 공유합니다.
- [ ] 발견된 이슈는 우선순위를 지정하고, 재발 방지를 위한 추가 테스트 시나리오를 업데이트합니다.
